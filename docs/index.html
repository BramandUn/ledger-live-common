▸ fetchHistodayRates(currencyOrCurrencies, fiatUnit, getLatestDayFetched)
src/api/countervalue.js

Fetch countervalue rates history for crypto currencies (or a currency) per day granularity.

fetchHistodayRates(currencyOrCurrencies: (Currency | Array<Currency>), fiatUnit: Unit, getLatestDayFetched: function (Currency): string?): Promise<CounterValuesPairing<Histoday>>
Parameters
currencyOrCurrencies ((Currency | Array<Currency>))
fiatUnit (Unit)
getLatestDayFetched (function (Currency): string?) will returns for a currency the latest day that is already loaded and don't need to be refreshed. It returns this in YYYY-MM-DD format. If data was never loaded OR all data needs to be refetched, you can return a falsy value. NOTE: This allow implementation to do two things:
  • don't fetch anything if current day is the latest
  • only fetch part of the data that are missing (patch)
Returns
Promise<CounterValuesPairing<Histoday>>: a subset of pairs with only the new data (regarding getLatestDayFetched)

GetPairRate

src/types/index.js

Synchronously lookup the history price of coin against a fiat. the returned countervalue is expressed in satoshis per cents so it can be multiplied by the value to convert. if Date is falsy, the current "latest" price is to be returned. the value 0 or any falsy returned value is means the countervalue is not available. It it up to GetPairHistory implementation to chose the date granularity to use. NB Ideally a get pair history implementation should fallback on 0 if before the timerange he knows, or "latest" after the timerange it knows

GetPairRate

Type: function (coinTicker: string, fiat: string): function (Date?): number?

Example
<any date before>    2018-03-01  2018-03-02  2018-03-03  <any date after>
0 0 0 0 0 0 0 0 0       9.65         9.22      8.77      latest latest latest ...
